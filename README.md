「这个世界上，总有些话，太珍贵，只值得写在信里，贴上邮票寄出。而她会等一个充满阳光的下午，慢慢拆开。」

---

## 🕊️ 项目哲学 (Philosophy)

在即时通讯（IM）过载的 2026 年，我们不需要另一个催促我们「秒回」的工具。**书信 (shuxin)** 是一场社交实验，旨在找回笔友时代的深情与体面。

- **连接的厚度**：非好友寄信默认「次日达」，人为制造物理距离感。
- **诚意的博弈**：每一封信都需要消耗一枚「数字邮票」，AI 会根据【诚意 + 质量】进行分拣。
- **注意力的主权**：默认不推送、不弹窗、无红点。爱看不看，默认不看也没事。
- **飞书的骨架**：借力飞书——文件永不过期、原图不压质、消息流稳定，但剥离办公的压迫感。

---

## ✨ 核心特性 (Features)

### 📬 赛博邮局 (The Cyber Post Office)

- **今日十封**：AI 每天只为你挑选 10 封最重要的信件，其余自动折叠。
- **昨日重现**：被折叠的好信，第二天依然有权重浮现，好诚意不被淹没。
- **火漆加密**：利用服务器端 AES 加密，除了收信人，连「邮差」也拆不开。

### 🎨 复古交互 (Retro Experience)

- **触感反馈**：模拟撕开信封、加盖火漆印的 Taptic Engine 震动。
- **信笺模板**：提供多种复古纸张与字体排版，让文字拥有重量。
- **AI 笔友**：支持 AI 角色主动发信，它也只有 10 张票，它在认真选你。

### 🛡️ 极简主义 (Minimalism)

- 只有四个菜单：**书信、消息、好友、我**。
- 拒绝朋友圈，拒绝视频号，拒绝一切与「见字如面」无关的噪音。

---

## 🛠️ 技术架构 (Tech Stack)

- **Frontend**: SwiftUI (原生 iOS 体验，追求极致丝滑)
- **Backend**: Node.js / Go (飞书 API 桥接网关)
- **Database**: Supabase / PostgreSQL (加密存储信件元数据)
- **Infrastructure**: Feishu/Lark Open API (消息流与文件底座)
