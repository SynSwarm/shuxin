# 书信 (shuxin) — 架构说明

说明 **仓库边界、运行时划分与信任边界**。实现细节随代码推进，记在 `STRUCTURE.md` 和后续 ADR 里。

---

## 1. 本仓库的定位（公开开源范围）

- **主体**：**SwiftUI / iOS 客户端** — 作为本仓库 **Apache 2.0 开源交付的核心**（愿景、产品说明与客户端源码对齐）。
- **网关（邮局后端）**：**不要求与本仓库一并开源**。  
  - 飞书 App Secret、用户令牌桥接、业务编排、加密与审计等多在服务端，适合与 **凭证与部署** 绑在一起管理，与 App Store 上的客户端 **解耦发布**。

因此：**把本仓库理解为「书信 iOS 应用 + 产品/架构文档」的公开主页**；客户端只通过 **HTTPS API 契约** 与网关对接。

---

## 2. 网关仓库（已决）

**已决**：邮局网关放在 **独立的私有 Git 仓库**（与本公开仓库分离）。

| 项 | 说明 |
|----|------|
| **私有仓** | 飞书 App Secret、数据库凭证、部署与 CI 与 iOS 证书/Provisioning 分离；公开克隆本仓库不会带入服务端实现。 |
| **协作方式** | iOS 与网关仅通过 **可配置的 API 基址**、**版本化 REST**（及后续若有的 WebSocket）协作；契约变更见下文「统一维护」。 |
| **本仓库** | **不包含** 网关源码目录；`.gitignore` 中 Node/后端段仅用于本地若有的脚本或工具，**不**作为「把网关藏在本机不提交」的替代方案。 |

曾讨论过的「同仓不同步开源」方案 **不作为本项目采用路径**，避免双远程与剔除目录的运维负担。

---

## 2.1 架构的统一维护（跨两仓）

**本仓库**里的 `ARCHITECTURE.md`、`STRUCTURE.md`、`PROGRESS.md` 和跨端契约摘要，和私仓一起约定 iOS 与网关共用的边界说法，避免各写各的：

- **信任边界、分层心智**（客户端 Presentation → Application → Domain → Infrastructure；网关 路由 → 应用服务 → 领域 → 适配器）两端对齐，命名可随栈微调但 **依赖方向一致**。
- **对外契约**（错误码、鉴权头、主要资源的 URL 与版本前缀）：优先在本仓 `docs/` 或 `ARCHITECTURE.md` 附录中维护 **摘要**；私有网关仓内保留 **OpenAPI 或等价清单** 作为真源，并与摘要同步。
- **重大变更**（加密策略、鉴权方式、破坏性 API）：在任一端落地前，先更新摘要 + **简短 ADR**（本仓 `docs/adr/` 或私仓 `docs/adr/`，至少一处可追溯，关键决策可镜像一份到本仓不含密钥的说明）。

**职责**：总架构师在 **两仓** 保持同一套原则；私仓中的目录树与网关专属的 `STRUCTURE.md` 由私仓维护，但需与本文 **不冲突**。

## 3. 端到端链路（逻辑视图）

```
[iOS 客户端] --HTTPS--> [网关：鉴权、业务、限流、审计]
                              |
                              +--> Feishu / Lark Open API
                              +--> Supabase / PostgreSQL（元数据等）
```

- **客户端**：只持有 **用户会话或短期令牌**（按平台最佳实践）；不嵌入 App Secret、不长驻服务端根密钥。
- **网关**：OAuth 与飞书能力、加密邮路、AI 分拣与成本控制、可观测性与合规点的 **责任边界**。

---

## 4. 信任边界（摘要）

| 位置 | 职责 |
|------|------|
| iOS | UI、本地状态、安全存储 Keychain 内 **用户级** 凭证；调用网关公开 API。 |
| 网关 | 验证身份、调用飞书、访问数据库、服务端加密/解密策略、审计日志。 |
| 飞书 | 消息/文件等 Open API；政策与配额以官方为准。 |

---

## 5. 与实现的同步约定

- 首次落地目录或大改分层时，更新 **`STRUCTURE.md`**。
- 阶段结论、已决/待办与下一刀，更新 **`PROGRESS.md`**。
- 引入新中间件、改加密或改网关形态时，在 **`docs/adr/`**（若已创建）或本文件追加简短 **ADR**：背景 / 选项 / 决定 / 风险 / 回滚。
